सत्य प्रतिबिम्ब — साझा पाठक फ़ीड (सभी विज़िटर्स को दिखे) — Netlify डिप्लॉय गाइड
================================================================================

क्या है
--------
अब "पाठकों की पोस्ट" सेक्शन में जो कोई भी विज़िटर पोस्ट डालेगा, वह एक साझा
सर्वर स्टोर (Netlify Blobs) में सेव होगी और बाद में साइट खोलने वाले हर
व्यक्ति को — किसी भी डिवाइस/ब्राउज़र पर — तुरंत दिखेगी। लाइक बटन भी सबके
लिए साझा है। संपादक पैनल से एडिट/डिलीट/प्रमोट भी सर्वर पर सेव होता है
(वही पुराना PIN: Satya@2026)।

फ़ोल्डर संरचना
---------------
- index.html                     मुख्य ऐप
- manifest.json                  PWA सेटिंग्स
- sw.js                          सर्विस वर्कर (network-first, हमेशा ताज़ा कोड)
- icon-192.png, icon-512.png     आइकन
- netlify.toml                   Netlify सेटिंग्स
- netlify/functions/posts.js     साझा पोस्ट का बैकएंड (पहले से bundle किया हुआ)

════════════════════════════════════════════════════════════════
✅ सुझाया गया तरीका — GitHub + Netlify (सिर्फ़ यही अपनाएँ)
════════════════════════════════════════════════════════════════
यही एकमात्र तरीका है जिसमें Blobs (साझा स्टोरेज) बिना किसी टोकन/एक्स्ट्रा
सेटअप के अपने आप ठीक से काम करता है। drag & drop और manual token वाले
तरीकों में बार-बार दिक्कत आती है (Netlify की अपनी सीमा) — इसलिए वो तरीके
अब न अपनाएँ।

**Step 1 — GitHub पर रिपॉज़िटरी बनाएं**
1) https://github.com पर लॉगिन करें (मुफ़्त अकाउंट काफी है)
2) ऊपर दाईं ओर "+" → "New repository"
3) कोई भी नाम दें (जैसे satya-pratibimb) → "Create repository"

**Step 2 — फ़ाइलें अपलोड करें**
1) नई बनी रिपॉज़िटरी में "uploading an existing file" लिंक पर क्लिक करें
   (या "Add file" → "Upload files")
2) इस पूरे फ़ोल्डर की सभी फ़ाइलें और फ़ोल्डर एक साथ खींचकर छोड़ें —
   index.html, manifest.json, sw.js, icon-192.png, icon-512.png,
   netlify.toml, और पूरा "netlify" फ़ोल्डर (उसके अंदर functions/posts.js)
3) नीचे "Commit changes" दबाएँ 

**Step 3 — Netlify से जोड़ें**
1) https://app.netlify.com पर लॉगिन करें
2) "Add new site" → "Import an existing project"
3) "Deploy with GitHub" चुनें → अपनी नई रिपॉज़िटरी (satya-pratibimb) चुनें
4) Build settings में कुछ भी बदलने की ज़रूरत नहीं — खाली छोड़ें या डिफ़ॉल्ट
   रहने दें (netlify.toml पहले से publish="." और functions पाथ बता देता है)
5) "Deploy site" दबाएँ — 30-60 सेकंड में साइट लाइव हो जाएगी

**बस इतना ही!** अब कोई Personal Access Token, कोई BLOBS_SITE_ID/BLOBS_TOKEN
जोड़ने की ज़रूरत नहीं। भविष्य में कुछ भी बदलना हो तो GitHub पर फ़ाइल एडिट
करें → Netlify अपने आप दोबारा डिप्लॉय कर देगा।

डिप्लॉय के बाद जाँच कैसे करें
-------------------------------
1) यह लिंक ब्राउज़र में खोलें: https://आपकी-साइट.netlify.app/.netlify/functions/posts
   → `{"posts":[]}` दिखना चाहिए, कोई error नहीं
2) साइट पर जाकर एक टेस्ट पोस्ट डालें
3) किसी दूसरे डिवाइस/नेटवर्क से साइट खोलकर देखें — वही पोस्ट वहाँ भी दिखे

सीमाएँ
-------
- फ़ोटो ~650KB से बड़ी अस्वीकार होगी
- नई से नई 300 पोस्ट रखी जाती हैं, पुरानी अपने आप हट जाती हैं
- कोई spam-रोधी सुरक्षा (reCAPTCHA आदि) शामिल नहीं है

────────────────────────────────────────────────────────────────
वैकल्पिक तरीके (सिर्फ़ अगर GitHub इस्तेमाल न कर सकें)
────────────────────────────────────────────────────────────────
Netlify CLI: टर्मिनल में इस फ़ोल्डर के अंदर जाकर — 
  npx netlify-cli login
  npx netlify-cli link      (मौजूदा साइट से जोड़ें)
  npx netlify-cli deploy --prod
यह भी बिना manual token के आमतौर पर सही तरीके से Blobs जोड़ देता है।

drag & drop से बचें — अगर फिर भी करना पड़े और "स्टोर उपलब्ध नहीं" वाली एरर
आए, तो Netlify Dashboard → Site configuration → Environment variables में
BLOBS_SITE_ID (Site details में मिलेगा) और BLOBS_TOKEN (User settings →
Applications → Personal access token से) जोड़ें — पर ध्यान रहे, कुछ
अकाउंट्स पर token बनाना "upgrade" मांगता है; ऐसे में GitHub तरीका ही
इस्तेमाल करें, वहाँ यह दिक्कत नहीं आती।
